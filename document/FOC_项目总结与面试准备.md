## 二、FOC 核心原理

### 2.1 坐标变换链

$$\text{三相电流 } I_a,I_b \xrightarrow{\text{Clarke}} I_\alpha, I_\beta \xrightarrow{\text{Park}} I_d, I_q$$

### Clarke 变换（幅值不变形式）

$$I_\alpha = I_a$$

$$I_\beta = \frac{I_a + 2I_b}{\sqrt{3}}$$

### Park 变换

$$I_d = I_\alpha\cos\theta + I_\beta\sin\theta$$

$$I_q = -I_\alpha\sin\theta + I_\beta\cos\theta$$

### 逆 Park 变换

$$V_\alpha = V_d\cos\theta - V_q\sin\theta$$

$$V_\beta = V_d\sin\theta + V_q\cos\theta$$

### SVPWM（七段式）

将 $V_\alpha, V_\beta$ 映射到六个基本电压矢量扇区，计算 $T_1, T_2, T_0$ 时间，输出三路互补 PWM 占空比 duty ∈ [0, 1]。

> **性能**（Cortex-M4 @ 170MHz, CMSIS-DSP 加速）：
> - `arm_sin_cos_f32` ×2：~0.3µs（比 sinf/cosf 快 ~5×，输入为**角度**非弧度）
> - 坐标变换 ×3：~1.0µs
> - SVPWM：~1.5µs
> - **合计 ~2.8µs**（50µs PWM 周期 < 6%）

### 2.2 Id=0 控制策略

- **Id 环**：目标 0A，控制 d 轴励磁分量为零。表贴式 PMSM 的永磁体已提供转子磁场，无需额外励磁。
- **Iq 环**：目标为 q 轴电流，转矩 $T \propto I_q$。正 Iq = 正转矩（逆时针）。

### 2.3 三级串级 PI 架构

| 层级 | 频率 | 输入 | 输出 | 控制器类型 |
|------|------|------|------|-----------|
| **电流环** | 20kHz | Id/Iq 误差 (A) | Vd/Vq 电压 (V) | PI |
| **速度环** | 1kHz | RPM 误差 | Iq 指令 (A) | PI |
| **位置环** | 100Hz | 位置误差 (LSB) | RPM 指令 | **P（纯比例）** |

**串级优势**：每层 PI 只处理单一物理量，调试独立，内环带宽 > 外环带宽 10 倍以上。

### 2.4 PID 设计特点

#### 并行式结构

$$u = K_p(K_r \cdot sp - fb) + K_i\sum e + K_d\Delta e$$

#### 条件积分抗饱和（Conditional Anti-Windup）

```
if (输出在限幅范围内):
    integral += Ki × error
else:
    integral 冻结（不累加）
```

#### 微分低通滤波（IIR）

$$D[n] = \alpha \cdot (e[n] - e[n-1]) + (1-\alpha) \cdot D[n-1]$$

$\alpha = 0.1$，-20dB/dec 高频衰减，抑制微分噪声放大。

#### Kr 加权（Setpoint Weighting）

| 环路 | Kr | 原因 |
|------|-----|------|
| 电流环 | 1.0 | 标准 PID，电流指令突变是正常操作 |
| 速度环 | 1.0 | 标准 PID |
| 位置环 | 1.0 | 纯 P，Kr 不影响 |

## 四、关键经验教训

### 4.1 🔴 ABC 三相接线 —— 错误一次浪费一整天

**这是整个项目中最容易被忽视、但最致命的坑。**

**症状对照表**：

| 症状 | 大概率原因 |
|------|-----------|
| PI 怎么调都振荡、发散 | **先查相序**，不要盲目改参数 |
| 转速方向和预期相反 | 任意对调两相即可反转 |
| 开环能转、闭环不行 | 编码器方向或相序错 |
| Iq 为正但实际转矩反向 | 电流传感器方向或相序错 |
| 偶尔能转但效率极低 | 相序错，旋转磁场与转子不同步 |

**铁律**：调试 FOC 前，先用开环电压拖动确认三相线按 **A-B-C** 接对，确认转子旋转方向和磁场方向一致，再进闭环。我们被 ACB 接法坑了大半天，改回 ABC 瞬间安静。

### 4.2 编码器回绕（Wrap-Around）—— 位置环的死穴

**问题**：AS5048A 是 14 位编码器（0–16383），转子越过零点时 `raw_angle` 从 16383 跳到 0，位置 PID 看到 $-16383$ 的阶跃误差，输出满幅速度指令，电机永远停不下来。

**解决方案**：引入 `unwrapped_pos`（展开累计位置）：

```c
// 第1步：通过 delta 检测回绕方向，累加到展开位置
raw_delta = raw_now - raw_prev;
if (raw_delta >  8192) raw_delta -= 16384;  // 正向回绕（0←16383）
if (raw_delta < -8192) raw_delta += 16384;  // 反向回绕（16383←0）
unwrapped_pos += raw_delta;
raw_prev = raw_now;

// 第2步：定期回绕到 setpoint ±8192 范围，防止数值无限增长
float diff = unwrapped_pos - pid_pos.setpoint;
if (diff >  8192) unwrapped_pos -= 16384;
if (diff < -8192) unwrapped_pos += 16384;
```

**为什么第2步重要**：如果不做第2步，`unwrapped_pos` 持续累加会达到百万级，`float32`（23位尾数）精度从 0.002 LSB 退化到 >1 LSB，最终位置控制失效。

### 4.3 PI 积分饱和 → 振荡

**问题**：位置环使用 PI 时，积分项持续累积导致转子到达目标后"冲过头"，反复振荡。

**解决**：位置环改用**纯 P 控制**（$K_i=0$）。云台电机摩擦力极低，纯 P 的稳态误差（约几个 LSB ≈ 0.1°）完全可接受，且不会像 PI 那样振荡。

**推广**：不是所有场景都需要 I。对外环（位置/速度），纯 P 往往是更安全的选择。

### 4.4 电流环 PI 调参过程

| 阶段 | Kp | Ki | 现象 |
|------|----|----|------|
| 初始 | 4.0 | 4.0 | 剧烈振荡，Iq 发散 |
| 降低 Ki | 4.0 | 2.0 | 仍然振荡 |
| 继续降低 | 4.0 | 0.3 | **稳定**，Iq 跟踪好 |
| 最终 | 4.0 | 0.3 | 稳态 Iq=0.1A, Vq=3–4V |

**经验**：Ki 不是越大越好。电流环 Ki 量级应在 $K_p / 10 \sim K_p / 100$ 范围。

### 4.5 速度环 Kr=0.5 的隐蔽 Bug

速度环 Kr=0.5 时，实际比例项 = $K_p(0.5 \cdot sp - fb)$，有效 setpoint 被减半，导致实际转速只有目标的一半。改为 Kr=1.0 后转速才准。

**教训**：Kr 改变了 setpoint 的有效值，调试时如果不注意 Kr，会误以为 Kp 不对而盲目增大。

### 4.6 ADC 噪声滤波

原始电流噪声 ±0.2A → 加 IIR 滤波（$\alpha=0.3$）→ ±0.05A。

$$y[n] = 0.3 \cdot x[n] + 0.7 \cdot y[n-1]$$

电流环 PI 对噪声敏感，滤波后的干净信号使 Ki 可以设得更低而不振荡。

## 五、面试常问问题

### 基础理论

#### Q1: FOC 的基本原理是什么？

通过 Clarke/Park 变换将三相交流电机等效为直流电机控制——分别控制励磁电流（Id）和转矩电流（Iq），实现类似直流电机的解耦控制。核心思想：在旋转坐标系（dq 轴）中，交流量变成直流量。

#### Q2: Id=0 策略为什么有效？什么时候不适用？

表贴式 PMSM 的转子磁场由永磁体提供，无需 d 轴励磁。Id=0 时：
- 全部电流用于产生转矩 → 效率最高
- 没有弱磁效应 → 适合中低速

**不适用场景**：内置式 PMSM（IPM）需要 $I_d<0$ 利用磁阻转矩；高速时需要 $I_d<0$ 弱磁扩速。

#### Q3: Clarke 变换的"幅值不变"和"功率不变"有什么区别？

| 形式 | $I_\alpha$ | 变换前后 | 本项目 |
|------|-----------|---------|--------|
| 幅值不变 | $I_a$ | 相电流峰值相等 | ✅ 使用 |
| 功率不变 | $\sqrt{2/3} \cdot I_a$ | 功率相等 | ❌ |

幅值不变形式下，SVPWM 的最大不失真输出为 $V_{dc}/\sqrt{3}$（相电压峰值）。

#### Q4: SVPWM 相比 SPWM 的优势？

| | SPWM | SVPWM |
|------|------|--------|
| 母线电压利用率 | $V_{dc}/2$ | $V_{dc}/\sqrt{3}$ |
| 利用率提升 | — | **+15.5%** |
| 谐波 | 较大 | 更小 |
| 实现复杂度 | 简单 | 需扇区判断和时间计算 |

#### Q5: Park 变换中的 θ 是什么？怎么获取？

$\theta$ 是转子 d 轴与 A 相绕组轴线的电角度。通过编码器（AS5048A 14-bit）读取机械角度，乘以极对数得到电角度。上电时需校准编码器零点偏移（enc_offset）。

### 控制环路

#### Q6: 为什么用三级串级 PI 而不是单级？

| 层级 | 带宽要求 | 物理限制 |
|------|---------|---------|
| 电流环 | ~2kHz | 电气时间常数 $L/R \approx 50\mu s$ |
| 速度环 | ~100Hz | 机械惯性 |
| 位置环 | ~10Hz | 运动规划 |

串级架构的优势：
1. 每层只处理一个物理量，调试独立
2. 内环比外环快 10 倍以上，对外环来说内环近似"瞬态响应"
3. 内环的限幅自然保护外环不过流

#### Q7: 电流环 PI 参数怎么确定？

理论公式：$K_p \approx L \cdot \omega_c$（$\omega_c$ 为目标带宽），$K_i = K_p \cdot T_s / T_i$。

本项目的 GM3506：$L=281\mu H$，目标带宽 $\omega_c \approx 14000$ rad/s（约 2.2kHz），$K_p \approx 281\mu \times 14000 \approx 3.9 \approx 4.0$。实际 $K_p=4.0, K_i=0.3$ 经验证稳定。

#### Q8: 位置环为什么用纯 P 不用 I？

- **I 的本质**：消除稳态误差。但代价是积分累积导致过冲和振荡。
- **P 的优势**：到达目标后输出归零，不会"冲过头"。
- **取舍**：云台电机摩擦力极低，P 的稳态误差（几个 LSB）可接受。

如果需要零稳态误差但不想振荡，可以加**死区（Deadband）**：$|e| < threshold$ 时输出 0。

#### Q9: 积分抗饱和（Anti-Windup）怎么实现的？

本项目使用**条件积分**（Conditional Integration）：

```
if (output_min < output < output_max):
    integral += Ki × error    // 正常累加
else:
    // 输出已饱和，冻结积分器，防止"深饱和"
```

退饱和时的危害：如果积分器在饱和期间持续累加（深饱和），退饱和时需要同样长时间才能"放掉"积分，导致长时间振荡。

#### Q10: 微分项为什么加低通滤波？

纯微分 $K_d \cdot \frac{de}{dt}$ 对高频噪声极度敏感——噪声的导数可以非常大。IIR 低通滤波 $\alpha=0.1$ 衰减高频分量，使微分项在实际系统中可用。

### 编码器与位置控制

#### Q11: 编码器回绕问题怎么解决的？

14-bit 编码器范围 0–16383，物理上 0 和 16383 是相邻位置。解决方法：

1. **Delta 检测**：$|\Delta| > 8192$ 判定为回绕，修正 delta
2. **展开位置**：`unwrapped_pos += delta`，得到连续的累计位置
3. **定期归中**：将 `unwrapped_pos` 回绕到 setpoint ±8192 范围，防止 float32 精度退化

#### Q12: 编码器零点偏移怎么校准？

通 $I_d=0.5A$ 直流 → 定子产生固定方向磁场 → 转子永磁体被拉到与磁场对齐 → 此时转子 d 轴与 A 相绕组对齐，$\theta_{elec}=0$ → 读取当前 raw_angle 存储为 `enc_offset`。

### 工程实践

#### Q13: ADC 采样怎么处理的？

- **双 ADC 同步采样**（ADC1 + ADC2）：同时采样 Ia 和 Ib，避免相位误差
- **TIM1 TRGO 触发**：PWM 中心对齐时（V4 时刻）采样，避开开关噪声
- **IIR 低通滤波**：$\alpha=0.3$，噪声从 ±0.2A 降到 ±0.05A
- **Ic 计算**：$I_c = -(I_a + I_b)$（基尔霍夫电流定律）

#### Q14: TIM1/TIM6/TIM7 的时序如何设计？

| 定时器 | 频率 | 作用 | 触发关系 |
|--------|------|------|---------|
| TIM1 | 20kHz | 三路互补 PWM + 死区 | TRGO → ADC1/ADC2 |
| TIM6 | 1kHz | 速度环 ISR | 独立运行 |
| TIM7 | 100Hz | 位置环 ISR | 独立运行 |

- TIM1 中心对齐模式，V4 时刻触发 ADC（避开开关噪声）
- TIM6/TIM7 频率拉开 10 倍，保证速度环远快于位置环

#### Q15: DRV8313 故障保护怎么处理？

```c
// GPIO EXTI 下降沿中断
void HAL_GPIO_EXTI_Callback(uint16_t GPIO_Pin) {
    if (GPIO_Pin == nFAULT_Pin) {
        drv_tim_pwm_moe_off();              // 关断 PWM 输出
        HAL_GPIO_WritePin(EN_Port, EN_Pin, 0); // 拉低使能
    }
}
```

#### Q16: 为什么整个 FOC 链路用物理单位而非归一化？

- Cortex-M4 有硬件 FPU（单周期 `float`），无性能损失
- 物理单位调试直观：`"Id=0.3A"` vs `"Id=0.15pu"`
- 唯一归一化处：SVPWM 输出到 `duty ∈ [0, 1]`（PWM 硬件接口要求）
- PID 不归一化：Kp 有明确量纲（V/A），调参时物理意义清晰

#### Q17: 整个 FOC 从零到全功能经历了哪些步骤？

| 步骤 | 内容 | 状态 |
|------|------|:----:|
| 1 | 编码器读取 + 方向验证 | ✅ |
| 2 | ADC 电流采样 + IIR 滤波 | ✅ |
| 3 | 编码器校准（Id=0.5A 锁轴） | ✅ |
| 4 | 电流闭环（Id=0, Iq 跟踪） | ✅ |
